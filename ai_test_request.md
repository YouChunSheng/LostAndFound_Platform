## 功能描述：失物招领平台的搜索功能

### 功能概述
失物招领平台提供了一个多条件搜索功能，允许用户根据关键词、分类、丢失地点和日期范围等条件搜索失物信息。

### 输入参数
1. **keyword** (可选)
   - 类型：字符串
   - 说明：搜索关键词，会匹配失物的标题和描述字段
   - 约束：最大长度255字符

2. **category** (可选)
   - 类型：枚举字符串
   - 说明：失物分类
   - 有效值：["证件", "电子设备", "衣物", "书籍", "其他"]

3. **location** (可选)
   - 类型：字符串
   - 说明：丢失地点，使用模糊匹配
   - 约束：最大长度255字符

4. **dateFrom** (可选)
   - 类型：日期字符串
   - 说明：搜索的起始日期（包含）
   - 格式：YYYY-MM-DD

5. **dateTo** (可选)
   - 类型：日期字符串
   - 说明：搜索的结束日期（包含）
   - 格式：YYYY-MM-DD

### 输出结果
- 类型：LostItem对象列表
- 说明：符合所有搜索条件的失物信息列表，按创建时间倒序排列
- 特殊情况：
  - 无匹配结果时返回空列表
  - 参数无效时可能返回错误或忽略无效参数

### 处理逻辑
1. 基于SQL查询构建，基础查询为"SELECT * FROM lost_items WHERE 1=1"
2. 根据提供的参数动态添加WHERE条件：
   - keyword存在时：添加"AND (title LIKE ? OR description LIKE ?)"条件
   - category存在时：添加"AND category = ?"条件
   - location存在时：添加"AND lost_location LIKE ?"条件
   - dateFrom存在时：添加"AND DATE(lost_time) >= ?"条件
   - dateTo存在时：添加"AND DATE(lost_time) <= ?"条件
3. 添加"ORDER BY created_at DESC"排序
4. 执行查询并返回结果

### 特殊处理规则
1. 所有条件都是可选的，未提供的条件不会影响查询
2. 多个条件之间是AND关系
3. keyword使用模糊匹配（包含匹配）
4. location使用模糊匹配（包含匹配）
5. dateFrom和dateTo使用日期范围匹配（包含边界）
6. 结果按创建时间倒序排列
